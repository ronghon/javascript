git優勢:分散式管理
  1.大部份操作在本地端完成,不需要連結網路
  2.完整性保證
  3.儘可能添加數據而不是刪除或修改數據
  4.分支操作非常快捷流暢
  5.與linux命令全面兼容

架構:
  工作區-->暫存區-->本地庫-->github代碼托管中心:負責維護遠端庫
  code(add)->commit(提交)->push(團隊權限)  
                     pull<-
本地庫與遠端庫操作形式
  1.團隊內部協作
  2.跨團隊協作

本地庫初始化
mkdir name  創建name目錄夾
pwd 查詢當前目錄夾路徑
ll 或ls 查詢目錄 -al含隠藏目錄及檔案
git init 初始化倉庫
git add 增加至暫存區
